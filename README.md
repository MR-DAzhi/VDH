Video DownloadHelper (VDH) 

本教程通过 Python 自动化脚本修改插件核心逻辑，彻底绕过 HLS/MPD 视频下载的 **2 小时等待限制**。

---

## 🛠️ 准备工作
1. **安装 Python**：确保电脑已安装 Python 环境。
2. **关闭 IDM**：操作时建议关闭 IDM 插件，防止干扰 Chrome 原生下载流程。

## 📖 操作步骤

### 第一步：定位插件目录
1. 打开 Chrome 浏览器，地址栏输入 `chrome://version/` 并回车。
2. 找到 **“个人资料路径”** (Profile Path)，复制该路径。
3. 在文件资源管理器中打开该路径，进入 `Extensions` 文件夹。
4. 找到文件夹 `lmjnegcaeklhafolokijcfjliaokphfk` (这是 VDH 的唯一 ID)。
5. 进入版本号文件夹（例如 `10.2.40.2_0`）。

1. 将 `10.2.40.2_0` 文件夹完整复制到 **桌面**。
2. 进入桌面上的文件夹，找到 `service` 目录（里面存有 `main.js`）。

使用小技巧
一、 环境维持（核心逻辑）
严禁放回原路径：修改后的文件夹必须放在非系统目录（如 D 盘），绝不能放回 AppData\...\Extensions，否则会被 Chrome 自动修复并还原。

固定加载方式：仅通过 Chrome 的“加载解压的扩展程序”指向该文件夹。如果文件夹移动了位置，必须重新加载。

备份 main.js：确保存有修改成功的 main.js 副本，防止误操作丢失。

二、 下载实操技巧
无视报错：重启浏览器后报错虽消失，但后续可能偶尔弹出。只要下载任务能开始，红字报错完全可以忽略。

并发控制：建议同时下载 2-3 个视频。并发过多会导致 CPU 占用过高，可能导致 CoApp 崩溃或视频合并失败。

三、 避开服务端风控
灵活切 IP：虽然 crack_vdh.py 伪造了授权，但若服务器对单个 IP 流量监控过严导致 402，切换 VPN 节点依然是最简单有效的物理绕过手段。

拒绝更新：插件：不要点商店更新，保持本地文件夹挂载。

四、 脚本备用
crack_vdh.py 的作用：它是你的“一键手术刀”。只有当插件重装或 main.js 被意外还原时，才需要重新运行它。平时无需理会。
